# G40-9 — front-end recovery UX (prototype)

**Branch:** `g40-9-frontend-recovery-ux`
**Scope:** FRONT-END ONLY, demo-grade, Request app first (`Final/gopher-request.html`).
No Connect/Deals. No payments/persistence/matching/admin built — deferred to the
human-dev backend track per `Final/AGENTS.md` / `CLAUDE.md`.

## What this adds

When a Gopher cancels an active request, the Requestor gets a **banner** + a **3-option
recovery modal**, plus a **15-minute no-action auto-cancel**. All UI lives in the Request
app dashboard and reuses existing primitives (the `DASH_DATA`/`renderActiveRequests`
dashboard, the `.ca-overlay`/`.ca-modal` shell, the `.rate-*` rating overlay look, and the
"clone terms → new request" idea behind Request-Again FAST). Semantic `g40-*` classNames;
no Figma token binding.

- **Demo trigger** — a clearly-labelled "Simulate: Gopher cancelled" bar on the dashboard
  (Home / All Requests). In production the trigger comes from the Gopher (Go) app; there is
  **no** Go-app wiring here.
- **Banner (AC1)** — "Your Gopher cancelled this request", with a "Review your options" reopen.
- **Recovery modal (AC1)** — three options + the countdown:
  - **Assign New Gopher** — reposts the same terms under a **new GR-##### order #**, marks the
    original closed, shows a "Reposted · Live / broadcasting" card. No edits.
  - **Edit Request** — opens **pre-filled + editable** Request Details (details, cost of items,
    payment to Gopher, pickup, drop-off, notes); submit **closes the original** and reposts under
    a **new order #**. Built as a **distinct** action (`data-g40-recover="edit"`); it does **not**
    reuse or modify the G40-266 in-place `edit-request` action (opposite create-vs-update rule).
  - **Cancel Request** — opens the rating overlay for the **cancelling Gopher** (its id/name),
    no repost, marks the original closed.
- **15-min auto-cancel (AC5, demo)** — an in-page countdown labelled "demo — production needs a
  durable server-side job". On expiry the original is closed.
- Every path **closes the original** (moves it to the dashboard **Cancelled** bucket) and
  **logs the cancellation** to the console with the gopher id + timestamp.

## Deferred to the human-dev backend track

Tagged **`G40-9 BACKEND SEAM`** at each call site in `Final/gopher-request.html`:

- **Real Stripe reassignment** — reuse the existing manual-capture `PaymentIntent`, hold funds
  across close→repost, **no new authorization**. Ref: `repostService.js` + `escrow.js` +
  `authRefresh.js`.
- **Real order lifecycle + persistence + cancellation logging** (gopher id + timestamp) — here
  it's in-memory + `console.log`.
- **Matching / re-broadcast** into the marketplace — here "Broadcasting…" is a label.
- **The durable 15-min timer job** — here it's a client `setInterval`.
- **Admin › Orders "repost on the Requestor's behalf"** — no admin surface exists in the prototype.

## Current-app-style fallback

None of these screens existed in `Code/Final` yet (the canonical flow stops at "Submitted" and
does not cover post-submit lifecycle). All new UI was built to match the existing Request-app
dashboard conventions (`.ca-*` modal shell, `.rate-*` rating overlay, `req-card` list). No
"current live app style" fallback was needed beyond conforming to the existing Final patterns.

## Tests

No test runner exists in this static repo, so the logic tests (`repostService.js`) travel with
the **backend** ticket, not here. "Done" for this PR = the flow clicks through end-to-end, which
was verified in-browser:

- AC1 banner + modal open, target = first in-progress request (GR-00127 / Marcus B.).
- AC2 Assign → original removed, GR-00201 reposted "Reposted · Live", resolved banner, 1 log.
- AC3 Edit → pre-filled (details/cost/pay/pickup/drop-off), edited total, GR-00202 reposted,
  original closed, 1 log; G40-266 `edit-request` action confirmed untouched.
- AC4 Cancel → rating overlay for Marcus B., star repaint (5→3), original closed, **no repost**,
  2 logs (rating + cancellation).
- AC5 auto-cancel → firing the branch (clock jumped past the deadline) closes the original with
  the "Auto-cancelled" banner. Countdown verified live (15:00 → 14:58).
- Fresh session prunes recovery-added Cancelled rows and resets the demo order-# counter; no
  duplicate rows; zero console errors across the session.
